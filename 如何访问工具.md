# 🌐 如何访问你的工具

## 📍 工具位置

你的工具位于：**F:\ProReport-GenAI**

---

## 🚀 快速启动（3步完成）

### 步骤1：打开项目目录

在文件资源管理器中打开：
```
F:\ProReport-GenAI
```

或者在命令行中：
```bash
cd F:\ProReport-GenAI
```

### 步骤2：启动服务

在项目根目录运行：
```bash
npm run dev
```

这个命令会同时启动：
- ✅ **前端服务器**（端口 3000）
- ✅ **后端服务器**（端口 3001）

### 步骤3：打开浏览器访问

在浏览器地址栏输入：
```
http://localhost:3000
```

或者直接点击：
👉 **http://localhost:3000**

---

## 📱 访问地址

### 本地开发访问

- **前端界面**：http://localhost:3000
- **后端API**：http://localhost:3001/api

### 首次访问

1. 打开浏览器
2. 访问：`http://localhost:3000`
3. 你会看到登录界面
4. 如果是首次使用，点击"立即注册"创建账号
5. 注册后自动登录，进入应用主界面

## ⚙️ 启动前检查

### 1. 检查依赖是否安装

如果第一次运行，需要先安装依赖：

```bash
# 安装前端依赖
npm install

# 安装后端依赖
cd server
npm install
cd ..
```

### 2. 检查环境变量

确保 `server/.env` 文件存在并配置了：

```env
GEMINI_API_KEY=你的API密钥
JWT_SECRET=随机字符串（建议32字符以上）
PORT=3001
FRONTEND_URL=http://localhost:3000
NODE_ENV=development
DATA_PATH=./data
```

## 🛠️ 常见问题

### Q1: 提示"端口被占用"

**解决方法：**
- 关闭占用端口的程序
- 或修改 `vite.config.ts` 中的端口号（前端）
- 或修改 `server/.env` 中的 PORT（后端）

### Q2: 提示"找不到模块"

**解决方法：**
```bash
# 重新安装依赖
npm install
cd server && npm install && cd ..
```

### Q3: 无法连接后端

**解决方法：**
1. 确保后端已启动（检查终端是否有后端日志）
2. 检查 `server/.env` 配置是否正确
3. 访问 http://localhost:3001/api/health 验证后端

### Q4: 页面显示空白

**解决方法：**
1. 检查浏览器控制台是否有错误（F12）
2. 刷新页面（Ctrl+F5 强制刷新）
3. 清除浏览器缓存
